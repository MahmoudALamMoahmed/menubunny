

## خطة نظام الباقات (Subscription Plans)

### الفكرة العامة

نظام باقات بسيط: كل مطعم يبدأ بباقة مجانية محدودة، ويمكنه الترقية لباقة مدفوعة تُخصم من رصيد المحفظة الموجودة حالياً.

### الباقات المقترحة

```text
┌─────────────┬──────────────┬──────────────┬──────────────┐
│             │  مجانية      │  أساسية      │  احترافية    │
├─────────────┼──────────────┼──────────────┼──────────────┤
│ السعر/شهر   │  0 ج.م       │  99 ج.م      │  249 ج.م     │
│ الفئات      │  3           │  10          │  غير محدود   │
│ الأصناف     │  10          │  50          │  غير محدود   │
│ الفروع      │  1           │  3           │  10          │
│ الإضافات    │  5           │  20          │  غير محدود   │
│ استقبال طلبات│  واتساب فقط  │  واتساب+لوحة │  واتساب+لوحة │
│ موظفي فروع  │  ✗           │  ✓           │  ✓           │
│ التحليلات   │  ✗           │  ✓           │  ✓           │
└─────────────┴──────────────┴──────────────┴──────────────┘
```

> الأرقام والأسعار قابلة للتعديل بسهولة لاحقاً — مخزنة في جدول `plans`.

### هل سنواجه مشاكل في الكود الحالي؟

**لا مشاكل كبيرة.** الكود الحالي نظيف ومنظم. كل عمليات الإضافة تمر عبر `useAdminMutations.ts`، فنحتاج فقط إضافة **فحص الحدود قبل الـ insert** في كل mutation. التغييرات:

1. **Mutations** — إضافة فحص حدود قبل `insert` في: `useSaveCategory`, `useSaveMenuItem`, `useSaveBranch`, `useSaveExtra`
2. **صفحات الإدارة** — إظهار رسالة "وصلت للحد الأقصى، قم بالترقية" بدل زر الإضافة
3. **لا تغيير** في منطق الطلبات أو صفحة المطعم العامة

### الخطة التقنية

**1. جداول قاعدة البيانات (3 جداول جديدة)**

- **`plans`** — تعريف الباقات (name, price_monthly, max_categories, max_items, max_branches, max_extras, features jsonb)
- **`subscriptions`** — اشتراك المطعم الفعال (restaurant_id, plan_id, status, started_at, expires_at)
- **`subscription_transactions`** — سجل عمليات الخصم من المحفظة للاشتراك

**2. دالة قاعدة بيانات (Security Definer)**

- `get_restaurant_limits(restaurant_id)` — تُرجع حدود الباقة الحالية للمطعم (أو حدود المجانية إذا لم يكن مشتركاً)
- `subscribe_to_plan(restaurant_id, plan_id)` — تتحقق من الرصيد، تخصم من المحفظة، وتُنشئ/تُجدد الاشتراك — كلها في transaction واحد

**3. تغييرات الكود (Frontend)**

- **hook جديد: `useSubscription.ts`** — يجلب الباقة الحالية وحدودها
- **hook جديد: `useLimitsCheck.ts`** — يقارن العدد الحالي بالحد الأقصى ويُرجع `canAdd`
- **تعديل `useAdminMutations.ts`** — إضافة فحص الحدود قبل كل insert
- **صفحة جديدة: `Subscription.tsx`** — عرض الباقات + الاشتراك/الترقية (خصم من المحفظة)
- **تعديل `Dashboard.tsx`** — إضافة رابط لصفحة الباقات + عرض الباقة الحالية
- **تعديل صفحات الإدارة** — إظهار تنبيه عند الوصول للحد الأقصى

**4. تجديد تلقائي (اختياري - مرحلة لاحقة)**

- Cron job عبر Edge Function يتحقق يومياً من الاشتراكات المنتهية ويُجددها من المحفظة أو يُخفض للمجانية

### ترتيب التنفيذ

1. إنشاء جداول `plans` + `subscriptions` + `subscription_transactions` مع RLS
2. إنشاء دوال `get_restaurant_limits` و `subscribe_to_plan`
3. إدخال بيانات الباقات الافتراضية
4. إنشاء `useSubscription.ts` + `useLimitsCheck.ts`
5. إنشاء صفحة الباقات `Subscription.tsx`
6. تعديل mutations لفحص الحدود
7. تعديل صفحات الإدارة لإظهار حالة الباقة والتنبيهات
