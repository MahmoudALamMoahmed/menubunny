

## التشخيص

المشكلة واضحة: الميزات المدفوعة (التحليلات، موظفي الفروع، طلبات لوحة التحكم) **معروضة فقط** في صفحة الاشتراك لكن **غير مفروضة** في أي مكان بالتطبيق. كذلك عند انتهاء الاشتراك والعودة للمجانية، لا يوجد أي تجميد للبيانات الزائدة.

## الحل

### 1. فرض قيود الميزات (Features Enforcement)

**3 صفحات تحتاج حماية:**

| الصفحة | الميزة | السلوك عند عدم التوفر |
|--------|--------|----------------------|
| `Analytics.tsx` | `analytics` | عرض رسالة "هذه الميزة متاحة في الباقة الأساسية" + زر ترقية |
| `BranchesManagement.tsx` (قسم الموظفين) | `branch_staff` | إخفاء/تعطيل قسم إدارة موظفي الفروع |
| `BranchesManagement.tsx` (order_mode) | `dashboard_orders` | منع اختيار "لوحة التحكم" وإجبار "واتساب فقط" |
| `Dashboard.tsx` | الأزرار | تعطيل أزرار التقارير والطلبات مع رسالة توضيحية |

**الآلية:** استخدام `useRestaurantLimits` في كل صفحة وفحص `limits.features`.

### 2. تجميد البيانات الزائدة عند العودة للمجانية

**المنطق:** عند عرض الفروع/الأصناف، نرتبها بـ `display_order` ونعتبر فقط أول N عنصر "فعال" (حيث N = الحد الأقصى للباقة). الباقي يظهر بشكل مجمد (معطل بصرياً + لا يمكن التعديل عليه).

**التطبيق:**
- في `BranchesManagement.tsx`: الفروع بعد الحد → بطاقة رمادية + badge "مجمد" + أزرار التعديل/التفعيل معطلة
- في `MenuManagement.tsx`: الأصناف/الفئات/الإضافات بعد الحد → نفس المعاملة
- في صفحة المطعم العامة `Restaurant.tsx`: الفروع المجمدة لا تظهر للزوار (فقط أول N فرع)

**مهم:** التجميد يكون في الواجهة فقط (Frontend). البيانات تبقى محفوظة في قاعدة البيانات.

### 3. منع order_mode = dashboard في المجانية

في `BranchesManagement.tsx` عند حفظ الفرع: إذا `dashboard_orders = false`، نُجبر `order_mode = 'whatsapp'` ونعطل الخيار.

## الملفات المتأثرة

| الملف | التغيير |
|-------|---------|
| `src/pages/Analytics.tsx` | إضافة فحص `limits.features.analytics` — عرض صفحة ترقية بدلاً من التقارير |
| `src/pages/BranchesManagement.tsx` | تجميد الفروع الزائدة + تعطيل موظفي الفروع + منع dashboard order_mode |
| `src/pages/MenuManagement.tsx` | تجميد الأصناف/الفئات/الإضافات الزائدة |
| `src/pages/Dashboard.tsx` | تعطيل أزرار الميزات غير المتاحة مع tooltip |
| `src/pages/Restaurant.tsx` | إخفاء الفروع المجمدة من العرض العام |
| `src/pages/Orders.tsx` | إضافة فحص `dashboard_orders` — عرض رسالة ترقية إذا غير متاح |

### ملاحظة مهمة
لن نواجه مشاكل في التطبيق. التجميد يعمل بالكامل من الواجهة الأمامية — البيانات تبقى محفوظة ومتاحة فوراً عند الترقية مجدداً. لا حاجة لتعديل قاعدة البيانات.
