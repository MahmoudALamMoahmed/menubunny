## مراجعة تصميم طرق الدفع بعد الإضافة - المشاكل المكتشفة

### المقارنة بين مناطق التوصيل وطرق الدفع

**مناطق التوصيل (SortableAreaItem):**

- `bg-gray-50 rounded-lg border` - خلفية رمادية موحدة
- عرض نظيف: صف واحد، النص على اليمين، الأزرار على اليسار
- `p-3` padding متساوي

**طرق الدفع (SortablePaymentMethod):**

- `bg-muted/40 rounded-lg border p-3` - خلفية مختلفة (muted بدل gray-50)
- حقول إدخال (inputs) مكدسة عمودياً مع grip handle + switch + زر حذف

### المشاكل التصميمية المكتشفة:

1. **عدم تناسق لون الخلفية**: مناطق التوصيل تستخدم `bg-gray-50` بينما طرق الدفع تستخدم `bg-muted/40` — يجب توحيدهما على `bg-gray-50`.
2. **محاذاة العناصر غير متسقة**: زر الـ grip handle وزر الحذف والـ switch كلها على `items-start` مما يجعلها غير متمحورة عمودياً بشكل جيد مع الحقول — يجب ضبط `mt` بشكل أدق أو استخدام محاذاة أفضل.
3. **زر الحذف وswitch التفعيل متلاصقان**: لا يوجد فاصل واضح بينهما، مما قد يسبب ضغط خاطئ على الموبايل.
4. **حقول الإدخال بدون labels مرئية**: في مناطق التوصيل النص واضح مباشرة، لكن في طرق الدفع placeholder فقط بدون label — بعد الكتابة لا يعرف المستخدم أي حقل هو أي.  
5.واجعل تصميم المناطق بعدضافتها تكون مناسبة للغه العربيه

### الإصلاحات المقترحة:

**ملف واحد: `src/pages/BranchesManagement.tsx` — مكون `SortablePaymentMethod**`

- توحيد الخلفية: `bg-gray-50` بدل `bg-muted/40`
- إضافة labels صغيرة فوق كل input (اسم الطريقة / رقم الحساب) بحجم `text-xs text-muted-foreground`
- تحسين spacing بين زر الحذف والـ switch بإضافة `gap-2` مناسب
- ضبط محاذاة الـ grip handle عمودياً مع أول input
